# Screen Mapping

미래에셋증권, 키움증권, 한국투자증권 MTS에서 동일 데이터/동일 기능 화면을 서로 매핑하기 위한 기준 문서.

## 문서 목적

- 기준 화면(미래에셋) 1개를 정하고 타 MTS의 대응 메뉴를 빠르게 찾는다.
- 화면명과 함께 API 스펙(엔드포인트, TR/ID)을 같이 보관해 기능 동등성을 검증한다.
- 불확실한 항목은 신뢰도/근거를 분리해 후속 검증 대상으로 관리한다.

## 매핑 표 컬럼 정의

| 컬럼명 | 설명 | 예시 |
| --- | --- | --- |
| 도메인 | 기능 분류(시세/수급/주문 등) | 수급 |
| 기준 증권사 | 기준이 되는 MTS | 미래에셋증권 |
| 기준 화면명 | 기준 화면 이름 | 투자자별 매매동향 |
| 기준 MTS 메뉴 경로 | 기준 앱에서의 이동 경로 | M-STOCK > 종목상세 > 투자자 |
| 비교 증권사 | 매핑 대상 증권사 | 키움증권 |
| 비교 화면명(앱) | 비교 앱 내 대응 화면/탭 이름 | 투자자(일별/누적) |
| 비교 MTS 메뉴 경로 | 비교 앱에서의 이동 경로 | 영웅문S# > 종목상세 > 투자자 |
| API 대응명 | 코드베이스에 정의된 대응 API 이름 | 종목별 투자자매매동향(일별) |
| API 식별자 | TR ID 또는 API ID | FHPTJ04160001 / ka10058 |
| API URL | 내부 스펙의 URL | /uapi/... /api/... |
| 근거 | 근거 출처(코드/외부) | specs_request.py, 외부 검색 스니펫 |
| 신뢰도 | 높음/중간/낮음 | 높음 |
| 비고 | 확인 필요 사항 | 공식 MTS 도움말 경로 확인 필요 |

## 확인 필요 항목

- 미래에셋/키움/한국투자 각사 공식 MTS 도움말(앱 내 가이드) 기준 메뉴 경로 최종 확정
- 화면 단위 동등성 검증: 종목 기준/시장 기준/일별 누적 기준의 정확한 매칭 여부
